# Stitch Prompt 模板

复制以下模板到 Stitch MCP 对话，替换 `{占位符}`。生成后归档到 `prompts/`。

**通用约束（每条模板均适用）**：

- 不要生成无关营销落地页
- 设计服务于真实项目流程（有声书清洗流水线）
- UI 文案使用**简体中文**
- 技术栈实现侧为 React + Vite + TypeScript（设计阶段不限，但组件应可映射）
- 桌面优先，宽屏布局
- 默认 **浅色（light）** 主题，高对比度状态色

---

## Dashboard 设计模板

```
请为「audiobook-cleaner-lab」设计主控制台 Dashboard。

【页面目标】展示有声书项目状态、章节进度、最近流水线任务与失败重试入口。
【目标用户】个人有声书后期制作者，非技术运营人员。
【项目类型】原文 + ASR + LLM 机切 + 人工 Review + FFmpeg 导出。
【布局】顶栏（项目名 + mock/real_api 模式徽章）+ 左侧导航 + 主区卡片与表格。
【核心组件】
- 项目/书籍列表（章节数、完成百分比）
- 最近任务表（ASR/对齐/LLM/导出，状态、时间）
- 失败任务区（错误摘要、重试）
- 快捷按钮：进入 Review 审核台
【交互状态】加载中、空状态、失败高亮、hover 行操作。
【中文文案】项目控制台、最近任务、失败重试、进入审核、mock 模式、真实 API。
【主题】浅色，专业工具风格，非营销页。
【禁止】登录页、定价页、无关插画营销块。
【输出】可导出 HTML screen 与截图，便于 React 实现参考。
```

## Review Workbench 设计模板

```
请设计「有声书 Review 审核台」桌面 Web 界面。

【页面目标】对照原文、ASR 与 LLM 机切建议，人工确认删除/保留/不确定区间。
【目标用户】有声书剪辑审核员。
【项目类型】audiobook-cleaner-lab，wavesurfer 波形编辑。
【布局】三栏上区 + 底栏全宽波形：
- 左：原文文本（高亮当前 source_range）
- 中：ASR segment 列表（时间戳、alignment 色标）
- 右：模型建议（keep/delete/uncertain、confidence、确认/驳回按钮）
- 底：音频播放器 + 波形时间轴 + 区间色块（绿保留/红删除/黄不确定）
【核心组件】segment 选中态、低置信醒目提示、保存按钮、导出 dry-run、文件路径显示。
【交互状态】播放中、未保存、segment 联动滚动、切点 ±2s 试听。
【中文文案】原文、语音识别、机切建议、确认、驳回、标记不确定、保存校正、导出。
【主题】浅色，信息密度高，工具型 UI。
【禁止】多轨 DAW、音效库、移动端单列简化版。
【输出】HTML + 截图，标注组件分区便于映射 React。
```

## Visual Review 设计模板（音频波形版）

```
请设计「音频波形视觉审核台」，用于有声书章节的质量目视检查。

【页面目标】通过波形与色块快速发现异常删除区间与低置信切点。
【目标用户】有声书质检员。
【项目类型】audiobook-cleaner-lab。
【布局】上大波形（可缩放）+ 下为 segment 列表与统计侧栏。
【核心组件】
- 全章波形缩略图与 zoom
- 区间 overlay（绿/红/黄）
- 播放头与 segment 同步
- 统计：总时长、删除时长、uncertain 数量
- 操作：通过、驳回、标记待听、重新生成建议
【交互状态】缩放、拖拽播放头、选中区间高亮。
【中文文案】波形审核、低置信、待人工试听、通过、驳回。
【主题】浅色。
【禁止】视频播放器、图片 grid（本项目为音频）。
```

## Mobile Responsive 设计模板

```
请为 audiobook-cleaner-lab Review 页设计「移动端响应式」参考（低优先级）。

【页面目标】在小屏上只读查看章节状态与 segment 列表，编辑仍以桌面为主。
【目标用户】外出时快速查看进度。
【布局】单列：顶栏章节信息 → segment 折叠列表 → 简化播放控件（无完整波形编辑）。
【核心组件】章节标题、状态徽章、segment 文本、播放/暂停。
【交互状态】折叠展开、加载中。
【中文文案】章节审核（只读）、请在桌面完成编辑。
【主题】浅色。
【禁止】完整三栏+波形编辑（桌面已实现）。
【备注】本项目当前默认桌面优先；此模板供未来扩展。
```
